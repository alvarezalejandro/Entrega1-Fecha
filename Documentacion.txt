En una primera versión del programa para el parseo de los diferentes formatos intente usar una una clase Formato y 3 clases que heredaban de ella, teniendo asi los 3 formatos posibles. Cada formato tenia un atributo que contenia un string con su formato correspondiente y un metodo para saber si la fecha a parsear era de ese formato o no, y otro metodo para parsear la fecha luego de haber chequeado que sea del formato correcto. Al momento de implementarlo encontre problemas parseando las fechas y encontrando el formato de en la conversion flexible. Por lo tanto decidi eliminar la clase formato y tener dentro de Fecha 3 SimpleDateFormat con cada uno de los formatos posibles, y usar las regex para matchear el formato correspondiente con la fecha, ya que son faciles de usar y hacen el codigo simple de entender.
Para los test utilize Calendar para comparar las fechas Date luego de ser parseadas, con las fechas creadas mediante el uso de Calendar. En los casos erroneos espere recibir una ParseException.
